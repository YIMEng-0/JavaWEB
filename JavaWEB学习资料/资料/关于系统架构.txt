1、系统架构分类：
	- C/S 架构
	- B/S 架构

2、什么是C/S架构，你常见的C/S架构的应用有哪些？
	- Client / Server
	- 客户端 / 服务器端
	- QQ、阿里旺旺.....
	- C/S架构中，客户端是有指定的客户端软件支撑的。

3、什么是B/S架构，你常见的B/S架构的应用有哪些？
	- Browser / Server
	- 浏览器端 / 服务器端
	- 浏览器端也是一个客户端，只不过这个客户端软件无论是哪个应用，都是一个客户端软件，浏览器。
	- B/S架构是C/S架构的特殊形式。
	- 淘宝、京东、网上银行.....

4、C/S架构的优缺点：
	- 优点：
		* 大部分数据已集成到客户端软件上，有少量的数据需要从服务器端获取，
		所以C/S架构速度很快，用户体验好。C/S架构是一个胖客户端形式。

		* 因为客户端软件是一个特定的客户端，这个客户端只服务于当前某一个
		应用，所以通常C/S架构的系统，客户端是很漂亮的。

		* 相对安全些，少量的数据集成在服务器端，大部分数据分散在客户端上。
		....

	- 缺点：
		* 必须安装特定的客户端软件
		* 升级慢，每一个客户端软件都需要升级
		...

5、B/S架构的优缺点：
	- 优点：
		* 不需要安装特定的客户端软件，只需要操作系统中有一个浏览器即可。
		* 升级容易，只需要升级服务器端
	- 缺点：
		* 速度慢，因为浏览器中没有集成该应用的任何数据，所有的数据都需要
		从服务器端获取，即使宽带很高，速度还是没有C/S架构的速度快。
		* 由于客户端是一个通用的浏览器软件，只支持html css javascript
		所以界面没有C/S界面漂亮
		* 所有的数据都放在服务器端，服务器出问题之后，数据可能全部丢失，
		相对不安全。
	
	- 结论：
		企业内部使用的系统，界面要求不是很炫，要求升级方便，
		所以企业内部【线下】使用的系统一般都是要求基于B/S架构的。
		C/S架构的系统是偏于娱乐相关的系统的。
